EXPLICACIÓN: SEGURIDAD DE IDENTIFICADORES Y HERRAMIENTAS DE DESARROLLO (F12)

1. LA REALIDAD DEL LADO DEL CLIENTE
Es importante entender que **es imposible** impedir físicamente que un usuario modifique el código HTML o JavaScript en su propio navegador. 
Todo lo que se envía al navegador (precios, IDs, scripts) está bajo el control total del usuario. 
Bloquear la tecla F12 o el clic derecho son medidas muy débiles que cualquier usuario con mínimos conocimientos puede saltarse en segundos.

2. LA SOLUCIÓN REAL: VALIDACIÓN EN EL SERVIDOR (PHP)
La única forma segura de proteger tu tienda es asumiendo que el usuario VA a intentar engañarte.
NUNCA confíes en los datos que vienen del formulario HTML (como precios ocultos).

Cómo implementarlo:
- En tus formularios HTML, envía solo el ID del producto (`id_producto`).
- Cuando el usuario envíe el formulario (el `POST` a tu archivo PHP, por ejemplo en `saskia/index.php` o al añadir al carrito):
    1. Recibe el `id_producto`.
    2. NO recibas ni confíes en el precio que venga del HTML.
    3. Haces una consulta a tu base de datos usando ese `id_producto`.
    4. Si el ID no existe (porque lo inventaron), muestras un error.
    5. Si el ID existe, usas el precio REAL que está en tu base de datos, no el que envió el usuario.

3. TRUCO VISUAL (ALERTA) - JAVASCRIPT
Si aun así quieres que salte una alerta cuando alguien modifique el HTML (aunque recuerda, esto no es seguridad real, es solo disuasorio), puedes usar un `MutationObserver`.

Añade este script al final de tus páginas de producto:

<script>
document.addEventListener('DOMContentLoaded', function() {
    // Selecciona el elemento que contiene el ID o el input oculto
    // Supongamos que tu input es: <input type="hidden" id="prod-id" value="123">
    const targetNode = document.getElementById('prod-id');

    if (targetNode) {
        // Configuración: observar cambios en los atributos (como 'value')
        const config = { attributes: true };

        // Función que se ejecuta cuando se detecta un cambio
        const callback = function(mutationsList, observer) {
            for(const mutation of mutationsList) {
                if (mutation.type === 'attributes' && mutation.attributeName === 'value') {
                    alert('AVISO DE SEGURIDAD: Se ha detectado una manipulación del código. La operación será cancelada.');
                    // Opcional: Recargar la página para deshacer cambios
                    location.reload();
                }
            }
        };

        // Crear e iniciar el observador
        const observer = new MutationObserver(callback);
        observer.observe(targetNode, config);
    }
});
</script>

RESUMEN
No gastes tiempo intentando bloquear F12. Gasta tiempo asegurando que tu código PHP compruebe siempre en la base de datos que el ID es válido y que el precio corresponde a ese ID antes de procesar cualquier compra.
